## 一、问题背景

### 1.1 现状

- `wechat-publisher` 技能基于 `wenyan-cli`，只能发布传统**图文消息**（news）
- 微信公众号 2023 年升级了「图片消息」功能，呈现类似小红书的横滑浏览体验（3:4 比例）
- 「图片消息」更适合 PPT 配图式短文（html-ppt 截图直接发布）
- wenyan-cli 不支持 `article_type: newspic` 格式

### 1.2 核心需求

- 将 html-ppt 产出的 slide 截图，以「图片消息」格式发布到公众号草稿箱
- 支持纯文本描述（正文内容）
- 本地 Skill 一键调用

## 二、微信 API 方案

### 2.1 图片消息 API 流程

微信公众号 `draft/add` 接口已支持图片消息。

**请求体关键字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `article_type` | string | 否 | `"newspic"` 表示图片消息，不填默认图文消息 |
| `title` | string | 是 | 标题，最长 32 字 |
| `author` | string | 否 | 作者，最长 16 字 |
| `content` | string | 是 | 正文内容，纯文本+部分特殊功能标签，< 2万字符，< 1M |
| `digest` | string | 否 | 摘要，最长 128 字 |
| `thumb_media_id` | string | 否 | 图文消息时必填，图片消息时不需要 |
| `image_info` | object | 是（图片消息） | 图片列表，最多 20 张 |
| `image_info.image_list` | array | 是 | `[{ image_media_id: "永久MediaID" }]`，首张为封面 |
| `need_open_comment` | number | 否 | 0 关闭(默认)，1 打开评论 |
| `only_fans_can_comment` | number | 否 | 0 所有人(默认)，1 仅粉丝 |

### 2.2 完整发布流程

```
1. 获取 access_token
   POST https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=APPID&secret=***

2. 上传图片为永久素材（获取 image_media_id）
   POST https://api.weixin.qq.com/cgi-bin/material/add_material?access_token=***&type=image
   (multipart/form-data, 每张图片单独上传)

3. 创建图片消息草稿
   POST https://api.weixin.qq.com/cgi-bin/draft/add?access_token=***
   {
     "articles": [{
       "article_type": "newspic",
       "title": "标题",
       "author": "叶帕奇",
       "content": "正文描述文字",
       "digest": "摘要",
       "image_info": {
         "image_list": [
           { "image_media_id": "图片1的MediaID" },
           { "image_media_id": "图片2的MediaID" }
         ]
       }
     }]
   }

4. 发布（可选，或手动在后台确认）
   POST https://api.weixin.qq.com/cgi-bin/freepublish/submit?access_token=***
   { "media_id": "草稿返回的media_id" }
```

### 2.3 图片要求

- 格式：PNG / JPEG / JPG / GIF
- 大小上限：10MB（单张）
- 数量上限：20 张（API 限制，PC 端 UI 限制 9 张）
- 展示比例：3:4（与 html-ppt 竖屏截图完美匹配）

## 三、网络架构设计

### 3.1 问题：IP 白名单

微信公众号 API 要求**调用方 IP 必须在白名单内**。

- 本地电脑 IP 是动态的，不适合直接加白名单
- 2026-05-17 部署 wenyan-cli 时就因为 IP 白名单问题折腾了一整晚
- 需要一个**固定 IP 的中转服务器**作为 API 代理

### 3.2 架构方案

```
┌─────────────────┐         ┌──────────────────────┐         ┌──────────────────┐
│  本地 OpenClaw   │ ──HTTPS──▶  中转服务器（固定IP）  │ ──HTTPS──▶  微信公众号 API  │
│  Skill 调用      │         │  /api/wechat/*        │         │  api.weixin.qq.com │
└─────────────────┘         └──────────────────────┘         └──────────────────┘
                                    │
                              ┌─────┴─────┐
                              │ access_token缓存 │
                              │ 图片临时存储     │
                              │ 请求转发        │
                              └───────────────┘
```

### 3.3 中转服务器职责

1. **请求转发**：接收本地 Skill 的请求，转发到微信 API
2. **IP 白名单**：服务器固定 IP，在微信后台加一次白名单即可
3. **access_token 管理**：自动获取和缓存 token（2 小时有效期），Skill 不用管
4. **图片中转**：接收本地上传的图片，转发到微信素材接口
5. **错误处理**：统一处理微信 API 错误码，返回简洁的错误信息

### 3.4 中转服务器 API 设计

```
POST /api/wechat/token              → 获取/刷新 access_token
POST /api/wechat/upload-image       → 上传图片，返回 image_media_id
POST /api/wechat/create-draft       → 创建草稿（图文/图片消息）
POST /api/wechat/publish            → 发布草稿
GET  /api/wechat/drafts             → 查询草稿列表
```

### 3.5 技术选型建议

- **中转服务器**：轻量 HTTP 服务，Node.js (Express/Fastify) 或 Python (FastAPI)
- **部署**：VPS / 云函数（需有固定出口 IP）
- **安全**：中转 API 加 simple auth（API Key 或 Bearer Token），防止滥用

## 四、Skill 设计

### 4.1 Skill 名称

`wx-newspic`（小绿书发布）

### 4.2 Skill 输入

```yaml
title: "文章标题"           # 必填，最长 32 字
author: "叶帕奇"            # 可选
content: "正文描述"         # 必填，纯文本
images:                     # 必填，图片路径列表
  - "/path/to/slide-1.png"
  - "/path/to/slide-2.png"
digest: "摘要"              # 可选，最长 128 字
```

### 4.3 Skill 输出

- `media_id`（草稿 ID，可用于后续发布）
- `created_at`（草稿创建时间）
- `success`（是否成功）
- 错误信息（如有）

### 4.4 与现有 Skill 的关系

- 复用 `wechat-publisher` 的微信凭证（APP_ID / APP_SECRET）
- 复用 `wechat-short-ppt` 的短文内容结构
- 复用 `html-ppt` 的截图产出

---

## 五、注意事项

- 图片消息的 `content` 字段只支持纯文本+部分特殊标签，**不支持 HTML 富文本**
- `image_media_id` 必须是**永久素材**的 MediaID（不是临时素材）
- 中转服务器必须保证微信后台白名单 IP 与服务器出口 IP 一致
- access_token 有效期 2 小时，中转服务需自动续期

## 六、参考资料

### wenyan-cli 命令结构（可复用参考）

| 命令 | 说明 |
|------|------|
| `wenyan publish` | 发布图文消息（只支持 news，不支持 newspic） |
| `wenyan serve` | 启动 HTTP 中转服务（默认端口 3000，支持 --api-key 认证） |
| `wenyan credential` | 管理微信凭证（APP_ID / APP_SECRET） |
| `wenyan token` | 管理 access_token |

**可复用的设计**：
- `wenyan publish --server <url>` 模式（本地 CLI → 远程服务 → 微信 API）
- `wenyan serve --api-key ***` 的认证机制
- 凭证管理方式（`--app-id` / `.env` 文件）

**不复用的部分**：
- 发布逻辑（wenyan-cli 只支持 `article_type: news`，不支持 `newspic`）
- Markdown 转 HTML 渲染（小绿书只需要纯文本）
