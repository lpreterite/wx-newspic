# 文档管理规范

> Document Management Protocols

**所属目录**：`ai-engineering/guide/`
**文档状态**：草稿
**当前版本**：v0.2
**发布日期**：2026-04-04

---

## 1. 概述

本文档定义 AI 软件研发流程中各类文档的创建、更新、归档管理规范，确保文档在整个项目周期内保持准确性和可追溯性。

---

## 2. 文档生命周期

### 2.1 生命周期阶段

```
┌─────────────────────────────────────────────────────────────────┐
│                       文档生命周期                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   创建        评审        发布        维护        归档          │
│     │          │          │          │          │              │
│     ▼          ▼          ▼          ▼          ▼              │
│  ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐        │
│  │ 起草  │──►│ 审核  │──►│ 发布  │──►│ 更新  │──►│ 归档  │        │
│  └──────┘   └──────┘   └──────┘   └──────┘   └──────┘        │
│                 │                        │                       │
│                 ▼                        ▼                       │
│           人工确认                    变更记录                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 各阶段说明

| 阶段 | 说明 | 负责人 |
|------|------|--------|
| **创建** | 按模板创建文档，填写初始内容 | 相关 Agent 或人类 |
| **评审** | 提交给干系人评审，确认文档质量 | PM Agent 协调 |
| **发布** | 评审通过后正式发布到文档体系 | PM Agent |
| **维护** | 根据项目进展更新文档内容 | 相关 Agent 或人类 |
| **归档** | 项目结束后归档，供后续参考 | PM Agent |

## 3. 文档创建规范

### 3.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 目录 | kebab-case | `project-tasks/`, `risk-management/` |
| 文档 | kebab-case | `status.md`, `task-list.md` |
| 图片/资源 | kebab-case | `logo-dark.png`, `icon-home.svg` |

### 3.2 文件头模板

所有文档应在开头包含以下元信息：

```markdown
# 文档标题

> 简短描述

**所属目录**：`xxx/`
**文档状态**：草稿 | 评审中 | 已发布 | 已归档
**当前版本**：v0.1
**发布日期**：YYYY-MM-DD
**最后更新**：YYYY-MM-DD

---

## 1. 概述
...
```

### 3.3 版本管理

| 版本格式 | 说明 | 示例 |
|----------|------|------|
| v0.x | 草稿阶段 | v0.1, v0.2 |
| v1.x | 正式发布 | v1.0, v1.1 |
| v2.x | 重大修订 | v2.0 |

版本递增规则：
- 草稿阶段：v0.1 → v0.2 → v0.3（小幅修改）
- 正式发布：v1.0（首次发布）
- 后续更新：v1.1（小幅修改）、v2.0（重大变更）

## 4. 文档更新规范

### 4.1 更新触发条件

| 触发条件 | 说明 | 更新责任方 |
|----------|------|-----------|
| 里程碑完成 | 阶段转换时更新状态 | PM Agent |
| 任务状态变更 | 任务完成/开始时更新 | PM Agent |
| 风险状态变更 | 风险识别/解决/升级时 | PM Agent |
| 人工决策 | 重大决策做出后 | 决策者 |

### 4.2 更新频率

| 文档类型 | 更新频率 | 说明 |
|----------|----------|------|
| STATUS.md | 按需 | 项目状态变更时立即更新 |
| milestones.md | 按周 | 每周检查里程碑进展 |
| tasklist.md | 每日 | 每日站会前更新 |
| 周报 | 每周 | 每周五发布 |

### 4.3 变更记录

每次更新文档时，必须在文档末尾的修订记录中记录：

```markdown
## 修订记录

| 版本 | 日期 | 修订内容 | 修订人 |
|------|------|----------|--------|
| v0.2 | 2026-04-04 | 新增 xxx 内容 | PM Agent |
| v0.1 | 2026-04-03 | 初始版本 | PO Agent |
```

## 5. 文档评审规范

### 5.1 评审流程

```
创建文档 → 提交评审 → 评审通过 → 发布文档
                │
                ▼
           评审不通过
                │
                ▼
           返回修改
```

### 5.2 评审参与方

| Gate | 评审内容 | 评审参与方 |
|------|----------|-----------|
| Gate 1 | PRD | Product Owner, PO Agent |
| Gate 2 | 技术方案 | Product Manager, Developer Agent |
| Gate 2 | 用户方案 | Product Owner, UI/UX Agent |
| Gate 3 | 可使用软件验收 | Product Owner, PM Agent |
| Gate 4 | 可维护软件验收 | Product Owner, PM Agent |

### 5.3 评审要点

| 评审项 | 说明 |
|--------|------|
| **完整性** | 文档内容是否完整覆盖要求 |
| **准确性** | 内容是否准确无误 |
| **一致性** | 与其他文档是否一致 |
| **可执行性** | 描述是否可操作执行 |

## 6. 文档同步规范

### 6.1 同步触发词

AI Agent 在检测到以下关键词时，应立即检查并更新相关文档：

| 关键词 | 应检查更新的文档 |
|--------|-----------------|
| "完成"、"实现了"、"解决了" | STATUS.md |
| "开始 M2"、"进入下一阶段" | STATUS.md, milestones.md |
| "发现阻塞"、"阻塞项" | STATUS.md |
| "提交"、"commit" | tasklist.md |

### 6.2 同步执行流程

1. 检测到项目变更（git commit / 功能完成 / 里程碑达成）
2. 判断变更类型
3. 确定需要更新的文档列表
4. 执行文档更新
5. 记录变更日志

### 6.3 状态同步规则

详见 [../conventions.md#8](../conventions.md#8-文档同步规范)

## 7. 文档归档规范

### 7.1 归档条件

| 条件 | 说明 |
|------|------|
| 项目结束 | 项目正式上线并完成验收 |
| 阶段结束 | 里程碑阶段完成并归档 |
| 文档过时 | 被新版本替代 |

### 7.2 归档操作

1. 将文档状态改为"已归档"
2. 将文档移动到归档目录（如 `docs/archive/`）
3. 更新索引文档中的链接

## 8. 文档访问权限

### 8.1 权限层级

| 层级 | 说明 | 权限 |
|------|------|------|
| **公开** | 所有成员可读可写 | 所有人 |
| **团队** | 团队成员可读写，其他只读 | 团队成员 |
| **受限** | 指定人员可读写 | 指定人员 |

### 8.2 文档权限矩阵

| 文档 | 读取权限 | 写入权限 |
|------|----------|----------|
| ai-engineering/* | 公开 | PM Agent, 人工 PM |
| STATUS.md | 公开 | PM Agent |
| product/PRD.md | 团队 | PO Agent, Product Owner |
| project-tasks/W*/ | 团队 | PM Agent |
