# 关键文档说明

> Key Documents and Deliverables

**所属目录**：`ai-engineering/guide/`
**文档状态**：草稿
**当前版本**：v0.2
**发布日期**：2026-04-04

## 1. 概述

本文档定义 AI 软件研发流程中各阶段的关键文档（产出物），包括：
- 文档内容要求
- 验收标准
- 责任人
- 存放位置

---

## 2. 打磨阶段关键文档

### 2.1 PRD（产品需求文档）

| 属性 | 说明 |
|------|------|
| **责任人** | PO Agent（起草）、Product Owner（验收） |
| **参与 Gate** | Gate 1 |
| **存放位置** | `docs/product/PRD.md` |

#### 内容要求

| 内容 | 说明 | 验收标准 |
|------|------|----------|
| **用户故事** | 以用户视角描述功能需求 | 完整、清晰、可测试 |
| **用户线路图** | 用户使用产品的路径图 | 覆盖核心场景 |
| **验收标准** | 判断需求是否完成的准则 | 可测试、无歧义 |
| **原型图** | 产品界面原型设计 | 覆盖核心页面、符合需求 |

#### 用户故事模板

```markdown
## 用户故事：[功能名称]

### 背景
[为什么需要这个功能]

### 用户需求
[用户想要什么]

### 验收标准
- [ ] 标准 1
- [ ] 标准 2
- [ ] 标准 3

### 优先级
[Must / Should / Could / Won't]
```

### 2.2 技术方案

| 属性 | 说明 |
|------|------|
| **责任人** | PM Agent + Developer Agent（起草）、Product Manager（验收） |
| **参与 Gate** | Gate 2 |
| **存放位置** | `docs/engineering/` |

#### 内容要求

| 内容 | 说明 | 验收标准 |
|------|------|----------|
| **系统设计** | 架构设计、模块划分 | 清晰、可实现 |
| **技术选型** | 技术栈选择、依赖说明 | 合理、有依据 |
| **接口契约** | 模块间接口定义 | 完整、无歧义 |

### 2.3 用户方案

| 属性 | 说明 |
|------|------|
| **责任人** | UI/UX Agent（起草）、Product Owner（验收） |
| **参与 Gate** | Gate 2 |
| **存放位置** | `docs/design/` |

#### 内容要求

| 内容 | 说明 | 验收标准 |
|------|------|----------|
| **设计规范** | 组件库、颜色、字体、间距 | 完整、一致 |
| **设计稿** | UI 设计稿 | 覆盖所有状态 |
| **交互说明** | 组件状态、过渡动画、微交互 | 清晰、可实现 |

#### 设计交付物标准

| 交付物 | 说明 | 格式要求 |
|--------|------|----------|
| **设计稿** | 完整的 UI 设计稿 | Figma/Sketch/Webflow 链接 |
| **标注规范** | 间距、颜色、字体等规范说明 | 设计稿内嵌或独立文档 |
| **尺寸规格** | 响应式断点、适配说明 | 包含桌面/平板/手机 |
| **交互说明** | 组件状态、过渡动画、微交互 | 独立文档或设计稿注释 |
| **图标资源** | 图标源文件或图标库引用 | SVG/PNG + 名称映射 |
| **设计标注** | 开发必需的标注信息 | Zeplin/Figma 导出或独立文档 |

### 2.4 里程碑规划

| 属性 | 说明 |
|------|------|
| **责任人** | PO Agent + PM Agent（起草）、Product Owner（验收） |
| **参与 Gate** | Gate 2 |
| **存放位置** | `docs/project-management/milestones.md` |

#### 内容要求

| 内容 | 说明 | 验收标准 |
|------|------|----------|
| **阶段划分** | 各阶段的起止时间和目标 | 合理、可执行 |
| **交付物清单** | 各阶段需要产出的文档/代码 | 完整、无遗漏 |
| **验收标准** | 各阶段完成的判定条件 | 清晰、可测试 |

## 3. 执行阶段关键文档

| **存放位置** | `docs/project-tasks/W{nn}/tasklist.md` |

| 内容 | 说明 |
|------|------|

## 4. 交付阶段关键文档

### 4.1 验收报告

| 属性 | 说明 |
|------|------|
| **责任人** | Tester Agent（起草）、Product Owner（确认） |
| **参与 Gate** | Gate 4 |
| **存放位置** | `docs/project-tasks/W{nn}/review.md` |

#### 内容要求

| 内容 | 说明 |
|------|------|
| **测试范围** | 本次验收涵盖的功能 |
| **测试结果** | 通过/失败的测试用例 |
| **遗留问题** | 未解决的问题及影响 |
| **验收结论** | 是否可以上线 |

|------|------|

| **存放位置** | `docs/STATUS.md` |

#### 内容要求

| 内容 | 说明 | 更新频率 |

| **阻塞项** | 当前阻碍进度的问题 | 按需 |
| **最近更新** | 最近的变更记录 | 按需 |

### 5.2 周报

| 属性 | 说明 |
|------|------|

## 6. 文档验收要点

### 6.1 PRD 验收清单

- [ ] 用户故事覆盖所有核心功能
- [ ] 用户线路图包含主要使用路径
- [ ] 验收标准可测试、无歧义
- [ ] 优先级划分合理

### 6.2 技术方案验收清单

- [ ] 系统架构清晰合理
- [ ] 技术选型有充分依据
- [ ] 接口定义完整、无歧义
- [ ] 考虑了扩展性和维护性

### 6.3 用户方案验收清单

- [ ] 设计稿覆盖所有页面和状态
- [ ] 标注规范完整
- [ ] 交互说明清晰
- [ ] 符合设计系统规范
