# 场景化用户故事：公众号文章样式定制技能

> 跟踪 Issue：[#43](https://github.com/lpreterite/wx-newspic/issues/43)

## 角色描述
公众号内容创作者/运营者，负责公众号文章的内容创作与发布，关注内容呈现的品牌一致性。不具备或不精通 CSS 设计技能，希望通过简单的方式让文章有统一的视觉风格。

## 现状工作流
- **当前方式**：在编辑器中手工调整每篇文章的排版样式（标题样式、文首文尾引流区/品牌标识、引用样式等），或用三方工具生成后仍需手工修复
- **痛点**：从标题到引用到引流区，每个环节的样式手工调整都耗时；多次发布的同类型文章无法复用样式

## 用户故事
**As a** 公众号内容创作者
**I want** 为文章套用预定义的视觉风格（标题样式、引用样式、文首文尾模块、品牌标识等）
**So that** 每次发布时不用逐项手工调样式，专注内容创作

## 业务验收标准
**Given** 用户有一篇 Markdown 格式的文章内容
**When** 用户通过以下任一方式定义样式：
- 提供一篇参考公众号文章链接 → 技能提取其整体视觉调性并生成风格
- 未提供参考文章 → 技能按文章元素类型（标题、正文、引用、代码块、文首文尾模块等）逐步提问用户，收集足够参数后生成风格
**Then** 技能输出符合品牌调性的内联样式 HTML，可直接在微信公众号草稿箱中正常展示
**And** 若用户明确表示不满意或放弃，视为失败，终止并回退

**前置条件**：
- 用户有 Markdown 内容（不强制，可使用示例文章验收）

**质量门禁**（待调研，跟踪于独立 Issue）：
- 可读性门禁标准需单独调研后补充（含微信 CSS 白名单兼容性、wenyan-md/core 模板可读性基准、移动端阅读习惯适配）
